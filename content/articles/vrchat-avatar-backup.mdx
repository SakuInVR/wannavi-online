VRChatアバター改変で一番つらいのは、「何を変えたせいで壊れたのか分からない」状態です。

服を入れる、シェーダーを変える、Quest対応する、表情メニューを足す。どれも楽しい作業ですが、バックアップなしで一気に触ると、エラーが出たときに戻る場所がありません。

Unityプロジェクトのバックアップやアバター制作効率化を整理して、初心者向けに一番大事だと思ったのは「作業前に戻れるセーブポイントを作る」ことでした。

## 最低限バックアップするもの

Unityプロジェクトを丸ごとコピーすると、数GBになることがあります。もちろん丸ごと保存してもいいですが、軽く残すなら、まず次の3つを保存します。

| フォルダ | 役割 |
| --- | --- |
| Assets | アバター本体、素材、シーン、Prefabが入る |
| Packages | 使っているパッケージ情報が入る |
| ProjectSettings | Unityプロジェクトの設定が入る |

この3フォルダだけをZipにすると、バックアップ容量を大きく減らす例が紹介されていました。復元時はUnityがLibraryなどを再生成するので、初回起動に時間はかかりますが、保存容量はかなり軽くできます。

## バックアップのタイミング

バックアップは、作業が終わったあとではなく、壊す前に取ります。

おすすめのタイミングはこの4つです。

| タイミング | 理由 |
| --- | --- |
| アバターを初めて読み込んだ直後 | 元の状態に戻せるようにする |
| 服や髪を追加する前 | 着せ替え失敗から戻るため |
| シェーダーを変える前 | ピンク色・表示崩れ対策 |
| Quest対応や軽量化の前 | 削りすぎたときに戻すため |

特に「とりあえず全部入れてみる」は危ないです。SDK、シェーダー、衣装、ツール類を一気に入れると、エラーが出たときに原因候補が多すぎます。

## 名前の付け方

バックアップは、名前が分かりやすいほど役に立ちます。

```text
avatar-original-2026-05-19.zip
avatar-before-costume-2026-05-19.zip
avatar-before-quest-2026-05-19.zip
avatar-after-upload-ok-2026-05-19.zip
```

「最新版」「完成版」だけだと、あとから見たときに何が違うか分かりません。日付と作業内容を入れるだけで、復旧判断がかなり楽になります。

## Prefabも複製する

プロジェクト全体のバックアップとは別に、アバターPrefabの複製も作っておくと安全です。

たとえば、元のPrefabを残したまま、改変用に別名で複製します。

```text
Avatar_Original
Avatar_CostumeTest
Avatar_QuestTest
Avatar_MenuEdit
```

これをやると、服の位置調整やメニュー改変で失敗しても、元のPrefabを見ながら戻せます。初心者ほど、作業用のPrefabを分ける価値があります。

## インポート前に見ること

アバターや衣装のUnityPackageを入れる前に、インポート画面を一度見ます。ここで大事なのは、「全部入れる」が正解とは限らないことです。

| 見る項目 | 理由 |
| --- | --- |
| VRCSDKフォルダ | 既存SDKと競合する可能性がある |
| Shaderフォルダ | 既に同じシェーダーが入っている場合がある |
| DemoやSample | 不要なら入れなくてもいい |
| FBX/Prefab | アバター本体に必要 |

古いパッケージには、今のVCC環境と合わないSDKが含まれていることがあります。既存プロジェクトに古いSDKを上書きすると、アップロードボタンが消えたり、メニューが壊れたりすることがあります。

不安なら、新しい空プロジェクトで一度開いて中身を見るのが安全です。

## 1つ変えたら確認する

アバター改変は、まとめて進めるほど原因不明になりやすいです。

おすすめはこの順番です。

1. バックアップを取る
2. 1つだけ変更する
3. Unity上で見た目を確認する
4. Gesture Managerなどで表情やメニューを確認する
5. SDKの警告を見る
6. 問題なければ次の変更へ進む

この流れなら、壊れたときに「直前の変更」が原因だと分かります。戻せる場所があるだけで、改変の心理的な怖さはかなり下がります。

## 復元するときの注意

バックアップから戻すときは、Zipを展開してからVCCでAdd Existing Projectします。初回起動時にUnityがLibraryを再生成するため、しばらく時間がかかります。

ここで焦って強制終了すると、別のトラブルが増えることがあります。復元直後は、Unityが止まったように見えても少し待つのが安全です。

また、同じUnityバージョンで開くことも大事です。VCCが推奨するバージョンを使い、古いUnityで無理に開かないようにします。

## 今日からの最低ルール

アバター改変を始めるなら、今日からこの3つだけ守ればかなり安全になります。

1. 作業前にZipバックアップを取る
2. Prefabを複製してから触る
3. 1つ変更したら確認する

これだけで、失敗しても戻れる状態になります。改変は失敗しない人がうまくなるのではなく、失敗しても戻れる人がどんどん試せるようになります。
