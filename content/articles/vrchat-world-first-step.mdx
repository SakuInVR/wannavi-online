VRChatワールド制作の最初の目標は、「すごい世界を作る」ではなく、「自分で作った場所に入れる」まで行くことです。

ここを間違えると、いきなり巨大な街、凝ったギミック、綺麗なライティング、Udonの仕組み、BOOTH素材の大量導入まで全部やりたくなって、Unityの画面を見た瞬間に手が止まります。最初の1本目は、床があり、スポーン地点があり、椅子に座れて、鏡でアバターを確認できて、ペンで遊べる。これで十分に「ワールド」です。

この記事では、YouTubeで見た3本の制作手順をもとに、初心者が今日やるべき順番だけに絞ってまとめます。

## まず作るもの

最初に作るのは、次のような「小さな練習部屋」です。

| 要素 | 目的 |
| --- | --- |
| 床 | 立てる場所を作る |
| 壁か背景 | 空間の端を感じられるようにする |
| スポーン地点 | 入室した瞬間の位置を決める |
| 椅子 | 友達が来たときに座れる場所を作る |
| 鏡 | アバター確認ができる定番要素を置く |
| ペン | 触って遊べる要素を1つ入れる |

このくらいの部屋でも、実際にVRChatへ入るとかなり感動があります。自分の作った床に立って、自分の置いた椅子に座れるだけで、「あ、ワールドってこうやって生まれるんだ」が身体でわかります。

逆に、最初から見た目を盛りすぎる必要はありません。壁紙、照明、家具、BGM、ギミックは、アップロードまで通ったあとに足す方が安全です。

## 作業の順番

最初のワールドは、この順番で進めます。

1. VRChat Creator Companionを入れる
2. VCCからWorldプロジェクトを作る
3. VCCが指定するUnityでプロジェクトを開く
4. 床を置く
5. スポーン地点を置く
6. 椅子、鏡、ペンを1つずつ置く
7. Unity上で動作確認する
8. VRChat SDKでエラーを確認する
9. サムネイルを設定する
10. テストアップロードする

大事なのは、Unity Hubから適当にプロジェクトを作らないことです。VRChatは対応UnityバージョンやSDKの組み合わせが重要なので、VCCからWorldテンプレートを作る方が詰まりにくいです。

また、動画ではUnityのバージョンが古い場合もあります。記事を書いている時点では、動画内の画面と今のVCC画面が違うことがあります。判断基準は「動画で言っていた数字」ではなく、「VCCが現在推奨しているUnity」です。

## 最初に詰まりやすいところ

初心者が止まりやすいのは、だいたいこのあたりです。

| 詰まりポイント | 起きること | 対策 |
| --- | --- | --- |
| Unityバージョン違い | SDKが動かない、アップロードできない | VCC指定のUnityを使う |
| プロジェクト名に日本語を使う | 予想外のエラー原因になる | 半角英数字で名前を付ける |
| スポーン地点がない | ワールドに入った位置がおかしくなる | VRCWorldを必ず置く |
| 床にColliderがない | すり抜ける、落ちる | 床オブジェクトの当たり判定を確認する |
| 椅子が座れない | 見た目だけの椅子になる | VRC_Chair系のPrefabを使う |
| 鏡やペンが動かない | 期待したギミックにならない | Prefabの説明とSDK警告を見る |

特にスポーン地点は地味だけど重要です。部屋がどれだけ綺麗でも、入室した瞬間に床の下へ落ちたり、壁の中に出たりすると、その時点でワールド体験としては崩れます。

## BOOTH素材は使っていい

最初から全部をBlenderで作る必要はありません。むしろ最初は、配布されている床、部屋、家具、ペン、鏡などを組み合わせて、「配置して動かす」感覚を覚える方が伸びやすいです。

ただし、素材を入れるときは利用規約を必ず確認します。

| 確認すること | 理由 |
| --- | --- |
| 商用利用の可否 | 将来、収益化サイトや配信で使う可能性があるため |
| 改変の可否 | 色変更や配置調整をしていいか確認するため |
| 再配布の可否 | ワールドに含めて公開して問題ないか確認するため |
| Quest対応の有無 | Questユーザーも入れるワールドにしたい場合に重要 |

「無料素材だから何でもしていい」ではありません。ここはかなり大事です。ワールド制作は、技術だけでなく素材の扱い方もクリエイターとしての基礎になります。

## 見た目より先に通すもの

最初の部屋で優先するのは、見た目の完成度ではなく次の5つです。

1. ワールドに入れる
2. 床に立てる
3. 鏡が見える
4. 椅子に座れる
5. ペンなどの小さな遊びが動く

この5つが通ると、「ワールド制作の回路」ができます。次からは、床を変える、壁を変える、ライトを変える、家具を増やす、BGMを置く、写真スポットを作る、という足し算に進めます。

逆に、この5つが通る前にライトベイクやUdonギミックに入ると、何が原因で壊れているのか判断しづらくなります。最初のワールドでは、欲張りを一回しまっておくのが強いです。

## 今日やるならここまで

今日のゴールは、「公開できる名作ワールド」ではなく、「テストアップロードできる小部屋」です。

作業メモとしては、次の状態になれば合格です。

- VCCからWorldプロジェクトを作った
- Unityで床を置いた
- VRCWorldを置いてスポーン位置を決めた
- 椅子、鏡、ペンを1つずつ置いた
- SDKの警告を読んだ
- サムネイルを設定した
- テストアップロードまで進んだ

ここまで行けたら、もう「VRChatワールドを作りたい人」から「最初のワールドを作った人」に変わっています。次にやるべきことは、豪華な街を作ることではなく、この小部屋に「自分らしさ」を1つ足すことです。

たとえば、好きな色の照明にする。写真を飾る。机を置く。BGMを流す。友達が来たときに座る場所を2つに増やす。そういう小さい足し算の方が、長く続くワールド制作になります。
