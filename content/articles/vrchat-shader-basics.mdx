VRChatのアバターが思った通りに表示されないとき、原因の一つがシェーダーです。
PCではきれいに見えても、Questでは同じ見た目にならないことがあります。

シェーダー設定で初心者が最初に焦るのは、初心者が最初に焦るのは「アバターがピンクになった」「服の裏側が消えた」「Poiyomiの項目が多すぎる」の3つです。
どれも壊れたわけではなく、原因を順番に見れば戻せます。

## まず確認すること

見た目がおかしいときは、次の項目を確認します。

| 確認項目 | 理由 |
| --- | --- |
| 使っているシェーダー名 | 対応環境を調べるため |
| PC向けかQuest向けか | Questで表示できるか確認するため |
| マテリアル数 | どの部位に何が使われているか見るため |
| テクスチャサイズ | 容量や負荷を見るため |
| VRChat SDKの警告 | アップロード前の問題を拾うため |

見た目の問題は、設定を一つずつ切り分けると解決しやすいです。

特にピンク表示は、モデルが壊れたわけではなく、必要なシェーダーがプロジェクトに入っていないサインであることが多いです。
配布ページやREADMEに「Poiyomi」「lilToon」などの推奨シェーダーが書かれていないか先に確認します。

## 初心者がやりがちなこと

よくあるのは、配布アバターの設定をよく見ずに、別のシェーダーやマテリアルを上書きしてしまうケースです。
変更前にスクリーンショットやメモを残すだけでも、戻しやすくなります。

特に気をつけたいのは次の3つです。

- 全マテリアルを一括で変更する
- PC用とQuest用の区別を見ない
- 元の設定を保存せずに上書きする

一括変更は便利ですが、初心者のうちは原因が分からなくなりやすいです。
まずは1つのマテリアルだけ変えて、見た目を確認します。

## ピンクになったときの直し方

アバターが全身ピンクになったら、まず落ち着いて導入順序を見直します。
多くの場合、アバター本体より先に必要なシェーダーを入れる必要があります。

安全な順番はこの形です。

1. VRChat SDKを入れる
2. アバター作者が指定するシェーダーを入れる
3. 必要ならVRCFuryなどの補助ツールを入れる
4. 最後にアバター本体を入れる

すでにピンクになっている場合は、指定シェーダーを入れたあとにマテリアルを確認します。
それでも直らない場合、マテリアルごとにInspectorで正しいシェーダーを割り当てます。

マテリアルが多すぎて手作業がつらい場合は、プロジェクトを作り直して正しい順番で入れ直した方が早いこともあります。

## Poiyomiで最初に触る場所

PoiyomiはVRChatアバターでよく使われるPC向けシェーダーですが、設定項目が多いです。
初心者が最初から全部触る必要はありません。

まず見るのはこのあたりです。

| 項目 | 何を見るか |
| --- | --- |
| Main Texture | 肌や服のテクスチャが正しく入っているか |
| Rendering / Opaque / Cutout | 髪や透明部分が正しく表示されるか |
| Outline | 輪郭線を出すか |
| Matcap | 金属感やツヤを足すか |
| Emission | 発光させるか |
| Cull | 服の裏面が消えていないか |

服の内側やフードの裏が透明に見えるときは、Cull設定が原因のことがあります。
CullをOffにすると裏面が見えるようになる例が出ていました。

ただし、Poiyomiは基本的にPC版向けです。
Quest対応では使えない、または別のモバイル向けシェーダーに置き換える必要があります。

## Quest対応で見た目が変わる理由

Questでは、PC版と同じシェーダーや表現が使えないことがあります。
そのため、PCで透明感や発光がきれいに見えても、Questでは簡略化された見た目になる場合があります。

これは失敗ではなく、環境の制限です。
Quest対応では、完全再現よりも「破綻しない見た目」を先に目指します。

PC版ではPoiyomiやlilToonで質感を作り込み、Quest版ではVRChatのMobile系シェーダーへ置き換える、という考え方が現実的です。
PC版とQuest版の見た目が違うのは普通です。

## 安全な試し方

シェーダーを変える前に、PrefabとUnityプロジェクトをバックアップします。
そのうえで、1つの部位だけ変更して確認します。

安全な流れは次の通りです。

1. 元アバターを複製する
2. 変更するマテリアルを1つに絞る
3. シェーダーを変える
4. Unity上で見た目を見る
5. VRChat SDKの警告を見る
6. 問題なければ次の部位へ進む

この手順なら、見た目が崩れても原因を戻しやすくなります。

さらに、Play Mode中に変えた設定は停止すると戻ることがあります。
水面や発光などの動きを確認するためにPlay Modeを見るのは有効ですが、調整値を保存したいときは再生中かどうかに注意します。

## 今日やること

今日やるなら、シェーダーを盛る前にこの5つだけ確認します。

1. アバター配布ページの推奨シェーダーを読む
2. SDK、シェーダー、アバター本体の順に入っているか確認する
3. ピンク表示ならシェーダー不足を疑う
4. 服の裏面が消えるならCullを確認する
5. Quest版はPC版と別シェーダーになる前提で考える

シェーダーは怖い場所ではなく、見た目の担当をしている設定です。
一括で触らず、1マテリアルずつ見ればかなり安全に進められます。
