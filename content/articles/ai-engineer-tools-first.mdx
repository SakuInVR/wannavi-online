AI開発を始めるとき、最初からツールを増やしすぎると学習より設定で疲れます。
最初の目的は「すごい環境を作ること」ではなく、小さなAIアプリを1つ公開して、次の改善点が見える状態にすることです。

Git/GitHub入門、Next.js環境構築、CursorとNext.jsの実演動画を見て整理すると、最初は「書く」「動かす」「戻す」「公開する」の4つだけに絞ると進めやすくなります。
特にAI時代は、AIが速く直すぶん、壊れたときに戻せる環境がないとかなり怖いです。

## 最初に入れる4点セット

最初の開発環境は、次の4点で十分です。

| 役割 | 選ぶもの | 目的 |
| --- | --- | --- |
| 書く | AI対応エディタ | エラー調査と実装相談を短くする |
| 動かす | Node.jsまたはPython | 小さなAIアプリを動かす |
| 戻す | GitHub DesktopとGitHub | AIが壊した変更から戻れるようにする |
| 公開する | VercelやRender | 作ったものをURLで見せる |

このセットがあれば、チャットUI、要約ツール、タグ生成ツール、学習ログアプリのような小さな作品を作れます。
最初からクラウド、Docker、GPU、データベースを全部そろえる必要はありません。

## GitHub Desktopを先に入れる理由

非エンジニア向けGit/GitHub動画で、いちばん大事だったのは「GitはAIとの共同作業の命綱」という考え方です。
AIエディタは便利ですが、予想外のファイルを変えたり、動いていた部分を壊したりします。

そのとき、Gitがあると「さっきの状態に戻す」ができます。
これはGoogleドキュメントの変更履歴に近いです。誰が、いつ、何を変えたかが残り、必要なら戻せます。

最初はコマンドを全部覚えなくてOKです。
GitHub Desktopを入れて、まずはこの3つだけ触ります。

1. 小さな `test.md` を作る
2. 変更内容を見てコミットする
3. AIに少し直させて、差分を見て戻す

注意点は、最初の練習リポジトリをPrivateにすることです。
AI開発ではAPIキーを扱うので、`.env` をGitHubに上げない、`.gitignore` に入れる、という習慣もここで作ります。

## Next.jsは「localhostで見えた」まで行く

Next.js環境構築動画では、Node.jsを入れて、VS Codeを入れて、`npx create-next-app@latest` でプロジェクトを作り、`npm run dev` で `localhost:3000` を開く流れが実演されていました。

ここで初心者が詰まりやすいのは、コードそのものより順番です。

| 手順 | 確認すること |
| --- | --- |
| Node.jsを入れる | `node --version` で表示されるか |
| VS Codeを入れる | フォルダを開けるか |
| Next.jsを作る | 対話型の質問は基本デフォルトで進める |
| フォルダに移動する | `cd project-name` を忘れない |
| 開発サーバーを起動する | `npm run dev` が通るか |
| ブラウザで見る | `localhost:3000` に表示されるか |

最初の成功体験は、自分の書いた文字がブラウザに出ることです。
細かいReactやTypeScriptの理解は、そのあとで足せばいいです。

## まず作るならこの3パターン

ツール選びで迷ったら、作るものを先に決めます。初心者が最初に作りやすいのはこの3つです。

| 作るもの | 学べること | 記事化しやすいポイント |
| --- | --- | --- |
| 要約ツール | API呼び出し、入力欄、結果表示 | どんな文章で失敗したか |
| 学習ログ整理ツール | Markdown、タグ付け、保存 | 学習を続ける仕組み |
| ポートフォリオ診断ツール | プロンプト設計、UI改善 | 改善前後の比較 |

重要なのは、最初から完璧なAIサービスを作らないことです。
入力欄があり、ボタンを押すとAIの結果が出て、そのURLを人に見せられる。
まずはここまでで十分です。

## 買うものより先に決めること

最初に決めるべきなのは、高い教材ではなく「何を1つ作るか」です。
作るものが決まると、必要なツールも教材も自然に絞れます。

たとえば、Webアプリを作りたいならNext.jsとVercelが候補になります。
Pythonで分析寄りのものを作りたいなら、Python、仮想環境、ノートブックの理解が先です。
ローカルAIや自宅サーバーに寄せたいなら、GPUやDockerの話が出てきますが、それは最初の1本を公開してからでも遅くありません。

## 失敗しやすい買い方

最初に避けたいのは、目的が曖昧なまま有料教材や高いPCを買うことです。
買った直後はやる気が出ますが、作るものが決まっていないと、環境構築だけで止まりやすくなります。

特に初心者は、次の買い方に注意します。

- 「AIエンジニア完全講座」だけを買って、作る題材を決めない
- GPUやサーバーを先に買って、使うアプリがない
- 複数のAIエディタを同時に試して、どれも浅くなる
- データベースや認証を入れすぎて、最初の公開まで遠くなる

最初の投資は、学習を止めないための道具に絞るのが安全です。

CursorとNext.jsの実演動画では、自然言語で「この機能を追加したい」と指示し、出たエラーをそのままAIに読ませて修正していました。
この流れは強いですが、前提としてNode.js、GitHub、ローカル実行、デプロイ先の基礎が必要です。

つまり、AIエンジニアの最初の道具は「AIに全部任せる魔法」ではなく、AIが出したコードを動かし、壊れたら戻し、公開できる最低限の作業台です。

## 30日後の到達点

最初の30日で目指す状態は、次の3つです。

1. 小さなAIアプリが1つ公開されている
2. GitHubにコードが残っている
3. 作った過程を記事にできるメモがある

この3つがそろうと、学習そのものがポートフォリオになります。
Wanna Naviでは、単に「勉強した」で終わらせず、作ったもの、詰まったところ、選んだ道具まで記事として残す方針で進めます。
