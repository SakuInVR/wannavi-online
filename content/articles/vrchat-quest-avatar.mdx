VRChatのQuest対応は、単にAndroid向けにアップロードするだけでは終わりません。
シェーダー、容量、マテリアル、ポリゴン数など、PC版とは違う制限を意識する必要があります。

Quest対応動画を分析すると、つまずきの多くは「Unityの準備不足」「PC版を直接触る」「Blueprint IDの紐付け忘れ」「揺れものを残しすぎる」のどれかです。
PC版の見た目を完全にそのまま再現しようとすると詰まりやすいので、最初は「Questで破綻せず見える」ことを目標にするのが安全です。

## まず確認すること

Quest対応に入る前に、次の項目を確認します。

| 確認項目 | 見る理由 |
| --- | --- |
| Unityのバージョン | SDKとの相性を見るため |
| VRChat SDKの状態 | 警告やアップロード条件を見るため |
| アバターのポリゴン数 | 軽量化が必要か判断するため |
| マテリアル数 | 表示負荷や容量に関わるため |
| 使用シェーダー | Questで使えるか確認するため |
| Android Build Support | Android向けにビルドできるか確認するため |

ここを先に見るだけで、かなり原因を切り分けやすくなります。

特に、Unityを入れただけではQuest用ビルドができないことがあります。
Unity Hub側でAndroid Build Supportが入っているか確認します。動画によってはiOS Build Supportも一緒に入れて、スマホ対応まで見据える流れが紹介されていました。

## よくあるつまずき

PC用シェーダーをそのまま使っていると、Quest側で表示が崩れることがあります。
また、テクスチャが大きすぎると容量制限に引っかかりやすくなります。

よくあるつまずきはこのあたりです。

- Questで対応していないシェーダーを使っている
- マテリアルが多く、軽量化しづらい
- テクスチャサイズが大きい
- PC版とQuest版を同じPrefabで触ってしまう
- SDKの警告を読まずに進めてしまう
- Blueprint IDを紐付けず、別アバターとしてアップロードしてしまう
- AndroidへのSwitch Platform中に止まったと勘違いする

最初は、PC版とQuest版を分けて管理するのがおすすめです。

## 次にやること

まずはアバターを複製して、Quest対応用の別バージョンを作ります。
元データを守りながら軽量化できるので、失敗しても戻しやすくなります。

作業順は次の通りです。

1. 元アバターをバックアップする
2. PC版を一度アップロード済みか確認する
3. Quest対応用のPrefabまたはシーンを作る
4. VRCQuestToolsで変換の設定を始める
5. シェーダーをQuest向けに変える
6. PhysBoneを残す場所と削る場所に分ける
7. マテリアルとテクスチャを整理する
8. PC版のBlueprint IDをQuest版に貼る
9. AndroidへSwitch Platformする
10. SDKの警告を確認してアップロードする

一度に全部直そうとせず、見た目が崩れた箇所を1つずつ確認します。

## Blueprint IDを忘れない

PC版とQuest版を同じアバターとして扱うには、Blueprint IDの紐付けが重要です。
これを忘れると、PC版とQuest版がVRChat上で別アバターとして登録されます。

流れはこうです。

1. PC版アバターをアップロードする
2. PC版のBlueprint IDをコピーする
3. Quest変換後のアバター側に貼り付ける
4. Android向けにアップロードする

この手順で、PCユーザーにはPC版、QuestユーザーにはQuest版が見える状態に近づけます。

## 何を残して何を削るか

Quest対応の核心は、ツールを押すことだけではなく、何を削るかを決めることです。
動画では、PhysBoneを全部残すのではなく、ケモ耳、尻尾、髪など、アバターの印象に関わる場所を優先していました。

判断基準はこのあたりです。

| 残しやすいもの | 削りやすいもの |
| --- | --- |
| 顔まわりの揺れ | 細かいアクセサリー |
| 尻尾や耳など個性の核 | 服の細かい揺れ |
| 表情に関わる要素 | ギミックや重い演出 |

Quest版では、10MBの壁やパフォーマンスランクの制限に当たりやすいです。
全部残そうとするとアップロードできなかったり、Very Poorになったりします。
最初は「その子らしさが残る最低限」に絞るのが安全です。

## 完璧を目指しすぎない

Quest対応では、PC版と同じ見た目にすることが難しい場合があります。
影、透明表現、発光、細かい装飾などは、軽量化のために調整が必要になることがあります。

最初は、表情が動く、服が大きく崩れない、色が分かる、アップロードできる、という基本を優先します。
見た目を詰めるのは、そのあとで大丈夫です。

PC用シェーダーがQuest向けのモバイルシェーダーに変わると、質感が大きく変わることがあります。
これは失敗ではなく、Quest対応の仕様に近いです。
「PC版そのまま」ではなく、「Questでも見える別バージョン」を作る感覚で進めると心が折れにくいです。

## 今日やること

今日やるなら、変換作業に入る前にこの5つだけ確認します。

1. プロジェクトを複製して戻せる状態にする
2. Android Build Supportが入っているか見る
3. PC版をアップロード済みか確認する
4. VRCQuestToolsとAvatar Optimizerの導入状態を見る
5. 残したいPhysBoneを3つ以内で決める

ここまでできれば、Quest対応の事故はかなり減ります。
変換ボタンを押す前の準備が、いちばん大事です。
